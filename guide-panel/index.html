Panels and call-to-action

Stack thumbnail images

Individual product thumbnails are square. Stacks, however, take up more screen space in most breakpoints. In order to create a pleasing layout, stack thumbnails need to match the height of single product thumbnails.

Within these layouts, there are differences depending upon how many product need to be displayed.

Note that the container for the stack can be as wide as it needs to be, and will still match the height of a single product, as long as that product is no wider than col-3.

Limitations

Because heights are hard-coded within this layout, it will not work as expected within nested grid layouts (for example in the mega-menu). The values are calculated for columns as children of the container element.

Text repeated behind an image

Using dynamically-created SVGs lets text repeat behind an image, where the text is always the full width of the container.

Note that if the text within the SVG changes, then the viewBox and the y values within the SVG need to be adjusted to accomodate for the length of the text string. The CSS will remain the same, however.

Note that the SVG will remain locked to the bottom of its container, and will maintain a square aspect ratio, unless the viewBox is adjusted to make it match its parent.
