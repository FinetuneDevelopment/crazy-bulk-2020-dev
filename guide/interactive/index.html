Accordions

The below is the minimum functional markup required, to create an accordion:

  • .accordion
    • input.accordion__control - hidden with a sr-only class
    • label.accordion__trigger accordion__spacing - linked to the above checkbox
    • .accordion__details - containing the content of the accordian pane

Note that as long as the checkbox appears first in the markup, the other two elements can switch round, so the tirgger comes after the details, if required.

The input element can be set to either checkbox or radio. If the latter is used, then groups of accordions can be linked together using the name attribute and only one accordion pane will be open at any one time.

If an accordion__icon element is used inside the accordion__trigger accordion__spacing, this element will reflect the current state of the connected accordion pane by rotating.

This accordion only allows one pane to be visible at any one time.

Block scroll

Block scroll can be used to add unlimited items to a page, which scroll off to the right.

Scrollers will take the width of their container and by default, their children will be 270px wide.

Child elements can take a width of 100% if a block-scroll__child-wide class is used directly on them. This means that each child will fit exactly inside the scrolling container and will be replaced with a different child, when the user interacts with the left and right buttons.

 

 

 

 

 

 

 

 

 

 

 

Dialog

The markup for dialog boxes needs to live close to the ending body tag, for positioning reasons. The JavaScript also only expects one copy of that markup. This introduces two issues:

  1. How do we have more than one dialog box on the page?
  2. What if the content we want to have inside the dialog box is nested deep in the markup?

For these reasons, the copy for the dialog box can sit wherever it is required and is copied into the dialog box only once the user triggers the dialog box to spawn.

In order to spawn the dialog box, you'll need a button tag with the attribute data-js="dialog-open". It also takes two other data paramaters:

Attribute Use
data-contentid The unique ID of an element anywhere in the markup, which has the content inside. Note that this element should also have the attribute hidden
data-heading A string which represents the heading of the dialog box (this will appear in the markup, but not display)

This method copies a chunk of markup from one place in the DOM and pastes it into the dialog container, which is found close to the footer. This means that any IDs are duplicated.

In order for elements such as labels to still function as expected, all ID attributes and for attributes on label tags are changed, to add 1 onto the end of them. For example, forename will become forname1.

Flyout (small)

Sometimes, content needs to hide within a flyout panel, but only on mobile. This allows the content to appear normally in breakpoints above sm, but automatically hides on mobile, with the ability for the user to toggle it on and off.

Where two flyouts might overlap each other in awkward ways, if you link together the two input elements as radio buttons, this ensures that as soon as one opens, the other will close.

As there is no native functionality to deselect radio buttons, once they have first been selected, adding an attribute of data-js="label-toggle" onto the label element allows this to happen.

If there is only one flyout in the immediate area, a checkbox will be all that is required.
