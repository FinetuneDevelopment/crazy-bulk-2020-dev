Tertiary call-to-action buttons

These were originally designed to be how secondary call-to-action buttons looked, when placed against dark backgrouds, but unfortunately, limitations with inheritance and CSS prevent this from being possible.

Note that because the text colour of the buttons is white and the background colour of the button is transparent, care should be taken to only use this element in a context where it has sufficient contrast.

Transparent buttons

Transparent buttons are design to sit adjacent to more important functionality. Note that these elements take on a blue colour by default, but will revert to a higher contrast colour as soon as they are used inside a background colour class.

Link Span

Generic button class

The generic btn class can be useful when creating subtle controls, paired with an icon. Note that Safari really struggles with the concept of inline-block, so Media layouts are required, in order to get the icons to align correctly.

Note that Safari has an issue with combining inline text and inline-block elements. However, as long as you remove all new line whitespace inside the btn, it seems to render fine. Spaces are OK.
